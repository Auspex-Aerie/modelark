# Contributing to ModelArk

The contribution guide lives at [`contributing/contributions.md`](contributing/contributions.md).
It covers the development environment, standalone and browser tests, project conventions, and the
fork-PR workflow. By participating, you agree to the repository's
[`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).

For security-sensitive reports, do not open a public issue; follow [`SECURITY.md`](SECURITY.md).
Migration, destructive-drive, and restore tests must use fixtures or disposable data; the project
does not use a contributor's live archive as a test target.
